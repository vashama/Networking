What is network monitoring?
Network monitoring is the use of a system that constantly monitors a computer network for slow or failing components and that notifies the network administrator (via email, SMS or other alarms) in case of outages. It is part of network management. A network monitoring system monitors the network for problems caused by overloaded and/or crashed servers, network connections or other devices. Commonly measured metrics are response time, availability and uptime.
The server monitoring may be internal, i.e. web server software checks its status and notifies the owner if some services go down, and external, i.e. some web server monitoring companies check the services status with a certain frequency.
External monitoring is more reliable, as it keeps on working when the server completely goes down and also third-party application performance monitoring tools provide additional web server monitoring, alerting and reporting capabilities.
Network monitoring services usually have a number of servers around the globe - for example in America, Europe, Asia, Australia and other locations. By having multiple servers in different geographic locations, a monitoring service can determine if a Web server is available across different networks worldwide. The more the locations used, the more complete is the picture on network availability.
